# 30분 퀵스타트

Visual Story Kit을 처음 쓰는 사람이 30분 안에 한 편의 기획 초안을 만드는 흐름입니다.
이미지 생성부터 시작하지 말고, 브리프와 캐릭터 고정값을 먼저 만듭니다.

## 목표

30분 뒤에 아래 3가지를 갖는 것이 목표입니다.

- 짧은 브리프 1개
- 캐릭터 고정값 1개
- 4장, 6장, 또는 8장 카드 플랜 초안 1개

## 0분-5분: 브리프 쓰기

아래 질문에 한 줄씩 답합니다.

```text
Audience: 누가 볼 콘텐츠인가?
Goal: 보고 나서 어떤 감정이나 행동이 남아야 하는가?
Topic: 무엇에 대한 이야기인가?
Format: 몇 장으로 만들 것인가?
Tone: 웃김, 따뜻함, 정보형, 담백함 중 무엇에 가까운가?
Constraints: 반드시 피해야 할 것 또는 꼭 넣어야 할 것은 무엇인가?
```

좋은 브리프는 길지 않아도 됩니다. 대신 대상, 목적, 톤이 보여야 합니다.

## 5분-15분: 캐릭터 고정값 만들기

`character-system/character-pack-template.md`를 복사해 캐릭터 하나를 만듭니다.
처음에는 너무 많은 디테일을 쓰지 않습니다.

반드시 고정할 것:

- 얼굴형
- 눈 모양
- 헤어스타일
- 체형
- 기본 의상
- 대표 소품
- 색상 hex

중요한 규칙:

```text
캐릭터 외형은 카드마다 다시 설명하지 않습니다.
캐릭터 팩과 레퍼런스 이미지가 외형을 담당합니다.
```

## 15분-25분: 카드 플랜 만들기

`prompt-system/planning-prompt.md`를 사용해 브리프를 카드 플랜으로 바꿉니다.
처음에는 6장을 추천합니다.

6장 기본 흐름:

```text
1. Hook: 멈춰 보게 만드는 질문 또는 장면
2. Situation: 상황 설명
3. Problem: 감정 또는 문제
4. Turn: 반전, 발견, 관점 변화
5. Interpretation: 내 관점 또는 의미
6. Closing: 한 줄 결론 또는 질문
```

카드별로 반드시 하나의 메시지만 넣습니다.

## 25분-30분: 첫 이미지 프롬프트 만들기

승인된 카드 플랜에서 1번 카드만 골라 `prompt-system/image-prompt.md`에 넣습니다.
처음부터 전체 카드를 생성하지 않습니다.

확인할 것:

- 1080 x 1350 캔버스를 명시했는가
- 1080 x 1080 safe area 안에 핵심 요소를 넣도록 했는가
- 캐릭터 레퍼런스를 첨부할 준비가 되었는가
- 외형 설명이 카드 프롬프트에 중복으로 들어가지 않았는가

## 꼭 지킬 규칙 5개

1. 이미지는 마지막에 생성합니다.
2. 카드 하나에는 메시지 하나만 넣습니다.
3. 캐릭터 외형은 캐릭터 팩에서만 관리합니다.
4. 레퍼런스 이미지는 배경이 아니라 캐릭터 정체성만 참고하게 합니다.
5. 실패한 카드는 전체가 아니라 해당 카드만 재생성합니다.
