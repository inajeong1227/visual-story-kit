# 리뷰 체크리스트

단계마다 승인 게이트가 있습니다. 게이트를 통과하기 전에는 다음 단계로 가지 않습니다.

```text
게이트 1: 스토리 플랜 + 캐릭터 목록 승인 -> 카드 플랜 작성 가능
게이트 2: 카드 플랜(JSON) 승인            -> 레퍼런스 생성 가능
게이트 3: 캐릭터/장면 레퍼런스 승인        -> 카드 이미지 생성 가능
게이트 4: 카드 이미지 승인                -> 발행 준비 가능
```

## 게이트 2: 카드 플랜 검수

- [ ] 카드 수가 브리프와 일치하는가
- [ ] 한 카드에 메시지가 하나인가
- [ ] 승인 목록에 없는 캐릭터가 없는가
- [ ] visual_prompt에 캐릭터 외형 묘사가 섞여 있지 않은가
- [ ] 페이싱 검수를 통과했는가 (`story-system/pacing-guide.md`)

## 게이트 3: 레퍼런스 검수

- [ ] 정면 마스터가 프로필/팔레트와 일치하는가
- [ ] 4-view 검수를 통과했는가 (`character-system/four-view-reference-guide.md`)
- [ ] 반복 장면의 레퍼런스가 준비됐는가
- [ ] 캐릭터 팩의 승인 표가 모두 체크됐는가

## 게이트 4: 카드 이미지 검수

카드마다 확인합니다.

### 캐릭터 일관성

- [ ] 4-view 레퍼런스와 같은 사람으로 보이는가
- [ ] 헤어스타일 실루엣이 같은가
- [ ] 신체 비율(머리 크기, 키)이 같은가
- [ ] 팔레트 색이 유지되는가
- [ ] 나이대가 달라 보이지 않는가
- [ ] 레퍼런스의 배경/조명이 카드에 새어 들어오지 않았는가

### 텍스트 가독성

- [ ] 스마트폰 실제 크기에서 헤드라인이 읽히는가
- [ ] 말풍선이 카드당 2개 이하인가
- [ ] 텍스트가 캐릭터 얼굴을 가리지 않는가
- [ ] 오타/이상한 글자가 없는가 (AI 생성 텍스트는 특히 확인)

### 레이아웃 안전

- [ ] 핵심 요소가 1080 x 1080 안전 영역 안에 있는가
- [ ] 워터마크/로고가 없는가
- [ ] 손가락 수, 눈동자 등 파손된 디테일이 없는가

### 스토리 흐름

- [ ] 이 카드만 봐도 역할(hook/turn 등)이 느껴지는가
- [ ] 이전 카드에서 넘어왔을 때 자연스러운가
- [ ] 같은 장면의 캐릭터 좌우 배치가 유지되는가
- [ ] 다음 카드로 넘기고 싶은가

실패 카드는 전체 재생성이 아니라 `prompt-system/regeneration-prompt.md`로
해당 카드만 재생성합니다. 단, 캐릭터 일관성 실패가 3장 이상이면
카드가 아니라 4-view 레퍼런스를 의심합니다.

## 발행 전 최종 체크

- [ ] 카드 순서가 맞는가
- [ ] 1번 카드가 썸네일로서 멈춰 보게 만드는가
- [ ] 캡션과 CTA가 준비됐는가
- [ ] 해시태그/멘션 확인
- [ ] 전체를 스마트폰에서 실제 스와이프로 훑어봤는가
