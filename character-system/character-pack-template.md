# 캐릭터 팩 템플릿

캐릭터 팩은 한 캐릭터의 모든 고정값을 하나로 묶은 단위입니다.
텍스트 프로필 하나가 아니라, 프로필 + 팔레트 + 레퍼런스 이미지 세트가 모여야 한 캐릭터가 "고정"됩니다.

```text
캐릭터 팩 = 프로필 문서 + 컬러 팔레트 + 4-view 시트 + 표정 시트 + 포즈 시트
```

## 팩 디렉토리 규약

캐릭터마다 아래 구조로 저장합니다. 이후 모든 이미지 프롬프트는 이 경로의 파일을 레퍼런스로 첨부합니다.

```text
characters/
  {character_name}/
    profile.md              # 이 템플릿을 채운 문서
    references/
      four-view.png         # 정면 / 3/4 / 측면 / 후면 한 장
      expressions.png       # 표정 시트
      poses.png             # 포즈 시트
      master-front.png      # 승인된 정면 마스터 이미지
```

## 3. 컬러 팔레트 (hex 필수)

색은 서술("따뜻한 갈색")이 아니라 hex 코드로 고정합니다.
각도와 장면이 바뀌어도 색이 흔들리지 않게 하는 핵심 장치입니다.

- 머리색: `#______`
- 피부톤: `#______`
- 눈 색: `#______`
- 기본 의상 메인: `#______`
- 기본 의상 서브: `#______`
- 대표 소품: `#______`

## 4. 스타일 고정값

- 선 스타일:
- 선 두께:
- 명암 수준:
- 배경 밀도:

## 5. 레퍼런스 상태

| 항목 | 파일 | 승인 여부 |
| --- | --- | --- |
| 정면 마스터 | `references/master-front.png` | ☐ |
| 4-view 시트 | `references/four-view.png` | ☐ |
| 표정 시트 | `references/expressions.png` | ☐ |
| 포즈 시트 | `references/poses.png` | ☐ |

정면 마스터가 승인되기 전에는 4-view를 만들지 않습니다.
4-view가 승인되기 전에는 카드 이미지를 생성하지 않습니다.

## 6. 반드시 유지할 것

- 같은 얼굴형
- 같은 헤어스타일
- 같은 신체 비율
- 같은 팔레트
- 같은 작화 스타일

## 7. 바뀌어도 되는 것

- 표정
- 포즈
- 손동작
- 상황 소품

## 8. 반드시 피할 것

- 실사풍 렌더링
- 다른 헤어스타일
- 다른 나이대로 보이는 얼굴
- 레퍼런스 이미지의 배경/조명/분위기를 따라 하는 것

## 캐릭터 그룹 (2인 이상 등장 시)

여러 캐릭터가 한 카드에 나오면 각자의 팩을 합치지 말고, 그룹 문서를 따로 만듭니다.

```text
characters/
  _groups/
    {group_name}.md
```

그룹 문서에 적을 것:

- 포함 캐릭터: (각 팩 경로 나열)
- 키 차이: (누가 더 큰지, 머리 하나 차이 등 상대 비율)
- 관계 거리감: (붙어 있는지, 떨어져 있는지)
- 배치 기본값: (누가 왼쪽/오른쪽인지 고정하면 컷마다 안 흔들림)
- 팔레트 충돌 체크: (의상 색이 비슷해서 구분이 안 되면 여기서 조정)

그룹 카드 생성 시 프롬프트에는 각 캐릭터의 4-view를 모두 첨부하고,
캐릭터마다 "이 레퍼런스는 {이름}의 것"이라고 명시합니다.
